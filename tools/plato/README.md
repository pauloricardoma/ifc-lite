# Plato clash-math single source

The clash narrow-phase math (Vec3, Box3/AABB, triangle intersection) is written
once in [Plato](https://github.com/cdiggins/plato) and transpiled to both Rust
and TypeScript so the two backends can never drift.

## What lives here

- `clash_math.plato` -- the single source of truth for the clash math kernel.
- `scaffold.plato` -- primitive scaffolding the Plato compiler assumes
  (concepts + intrinsic types), trimmed from `demos/plato-src/geometry.plato`
  in `cdiggins/plato` (MIT).
- `scalar-codemod.mjs` -- deterministic AST post-pass over the generated
  TypeScript: rewrites scalar method dispatch to native operators and lifts the
  Number/Boolean prototype helpers into a module-scoped namespace, so the
  committed output touches no native prototypes.
- `flatten-codemod.mjs` -- phase 2 of the post-pass (invoked by
  scalar-codemod.mjs): symbolically inlines the pure Vec3/Box3 method bodies
  down to scalar arithmetic over tuple inputs (beta reduction + scalar
  replacement + hash-consed common-subexpression hoisting) and emits flat
  tuple-native kernels with zero per-call object allocation. These are what
  the packages/clash math adapters bind; end to end they benchmark faster
  than the old hand-written kernel.

The generated outputs are **committed** and are the source of truth for every
build:

- `rust/clash/src/generated/plato.rs`
- `packages/clash/src/math/generated/plato.g.ts`

**Consumers never need the .NET SDK or this toolchain.** Building, testing and
shipping ifc-lite reads only the committed files above. The generator runs
solely when `*.plato` changes, and a CI freshness gate proves the committed
outputs stay in sync (see below).

## Pins

Generation is pinned for reproducibility. The script clones these SHAs:

| Repo | SHA |
| --- | --- |
| [cdiggins/plato](https://github.com/cdiggins/plato) | `18538a4c1ddb5f7bea5bd135b5712b7625c22845` |
| `plato` submodule [ara3d/parakeet](https://github.com/ara3d/parakeet) | `1e385d19e1eba592595df1cc46ae948f0d419bfc` |
| [ara3d/ara3d-sdk](https://github.com/ara3d/ara3d-sdk) | `030c9350ef56f883e96bdeacf0f8a467ec7b21df` |

`Plato.CLI` references `..\..\..\ara3d-sdk`, so the checkouts must be laid out
as `<work>/ara3d-sdk` and `<work>/<subdir>/plato`. The generator arranges this
for you. It requires the **.NET 9 SDK** (`dotnet build -c Release`).

## Regenerate

```sh
# Regenerate + overwrite the committed outputs.
node scripts/generate-plato-clash.mjs

# CI gate: fail (exit 1) if the committed outputs are stale.
node scripts/generate-plato-clash.mjs --check
```

Env overrides (CI leaves them unset; local reuse of an existing checkout/SDK
avoids re-downloading):

- `PLATO_WORK_DIR` -- work/cache root for the checkouts
  (default: `node_modules/.cache/ifc-lite-plato`).
- `PLATO_SUBDIR` -- subdir under the work root that holds the `plato` checkout,
  present so the `..\..\..\ara3d-sdk` csproj reference resolves (default: `src`).
- `PLATO_DOTNET` -- `dotnet` executable (default: `dotnet` on `PATH`).
- `PLATO_RUST_OUT` / `PLATO_TS_OUT` -- override the final output paths.

## Generation details

Three things the script does that are easy to get wrong:

1. **The CLI exits 0 even on failure.** `Plato.CLI` returns exit code 0 in
   generation mode whether or not parsing/compilation succeeded. A silent
   failure yields a missing or stub output, not a nonzero exit. The script
   therefore verifies each raw output **exists and exceeds 5000 bytes** and
   hard-fails otherwise -- this size check is the only reliable success signal.

2. **The timestamp line is stripped.** The CLI prepends
   `// Autogenerated file: DO NOT EDIT` followed by a nondeterministic
   `// Created on <timestamp>` line. The script drops that two-line block and
   prepends the deterministic canonical header
   (`// Autogenerated from tools/plato/clash_math.plato - DO NOT EDIT ...`), so
   regeneration is byte-identical run to run.

3. **The TypeScript goes through the scalar codemod.** After the CLI and before
   header finalization, the TS is passed through `scalar-codemod.mjs`
   (`node tools/plato/scalar-codemod.mjs <rawInput> <output>`) to erase the
   Plato scalar wrapper structs to native primitives. The Rust output needs no
   such pass.

## CI freshness gate

`.github/workflows/test.yml` runs `node scripts/generate-plato-clash.mjs
--check` on a free runner whenever the `.plato` sources, the committed outputs,
or the generator change. A stale committed output fails the PR with a
regenerate hint. The job is path-filtered, so it is skipped (treated as OK) on
PRs that do not touch these files, exactly like the other filtered jobs.
