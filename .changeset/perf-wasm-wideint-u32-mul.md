"@ifc-lite/wasm": patch

Speed up the exact CSG kernel on wasm by ~9% (single-thread, CSG-dense model) with byte-identical output. The kernel's fixed-width predicate/arrangement arithmetic runs in `bnum` integers (I256/I512/I1024/I2048), whose schoolbook multiply accumulates each partial product in a u128 digit. wasm32 has no 128-bit multiply, so every partial product lowered to a `__multi3` software libcall, the dominant cost of the kernel on wasm. This vendors `bnum` 0.14.4 (`third_party/bnum`, wired via `[patch.crates-io]`) with one functional change: on `target_arch = "wasm32"` the multiply uses u32 digits, making each partial product a single `i64.mul` with no libcall. Native codegen is unchanged (native keeps u128 digits). The result is byte-identical because a fixed-width product, and whether it overflows the signed width, are independent of the accumulation digit width. Verified by a differential test (`rust/geometry/tests/wide_mul_bit_identity.rs`: independent u32-limb and u64-limb schoolbooks agree with each other and with `bnum::checked_mul` over a random plus adversarial corpus for every width), the `mesh_determinism` manifests passing with no re-pin, and an A/B wasm run whose mesh position/index/normal checksums are unchanged. Measured 3947ms to 3596ms of single-thread geometry compute on `advanced_model.ifc`.
